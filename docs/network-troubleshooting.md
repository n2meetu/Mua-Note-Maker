# 网络问题排查

## ✅ 确认服务正常

```bash
# 电脑上测试（成功）
curl http://192.168.3.239:3000/health
# 返回: {"status":"ok","timestamp":"2026-04-03T16:12:24.750Z"}

# 服务监听状态（正常）
netstat -an | grep 3000
# 返回: tcp4  0  0  *.3000  *.*  LISTEN
```

## ❌ 手机无法访问原因

### 1. 端口号输错了（最常见）

❌ 错误：`http://192.168.3.239:300/health`（少了最后一个0）
✅ 正确：`http://192.168.3.239:3000/health`

### 2. macOS 防火墙阻止

**检查方法：**
1. 打开"系统设置" → "隐私与安全性" → "防火墙"
2. 如果防火墙已开启，点击"选项"
3. 找到 `Node` 或点击 "+" 添加 Node

**或临时关闭防火墙（仅测试用）：**
- 关闭防火墙开关
- 测试完成后记得重新开启

### 3. 手机和电脑不在同一 Wi-Fi

**检查方法：**
- 手机：设置 → Wi-Fi → 查看当前连接的网络
- 电脑：系统设置 → Wi-Fi → 查看当前连接的网络
- 必须是**同一个 Wi-Fi 名称**

**常见问题：**
- 5G Wi-Fi vs 2.4G Wi-Fi（不同频段也是同一网络）
- 公司网络可能有 AP 隔离（设备间无法通信）

### 4. 路由器 AP 隔离

某些路由器（特别是公司网络）会阻止设备间通信。

**测试方法：**
- 在手机上 ping 电脑 IP（需要安装网络工具APP）
- 或用另一台电脑/平板测试是否能访问

## 🔧 临时解决方案（推荐）

### 方案1：使用 ngrok 内网穿透

```bash
# 安装 ngrok（如果未安装）
brew install ngrok

# 启动内网穿透
ngrok http 3000
```

会显示类似：
```
Forwarding  https://abc123.ngrok.io -> http://localhost:3000
```

修改小程序配置：
```javascript
// miniprogram/utils/config.js
const API_BASE_URL = 'https://abc123.ngrok.io'
```

**优点：**
- 不受防火墙限制
- 手机可以用 4G/5G 访问
- 支持 HTTPS（小程序正式版需要）

### 方案2：关闭 macOS 防火墙

仅用于开发测试，测试完记得开启。

### 方案3：使用手机热点

1. 手机开启热点
2. 电脑连接手机热点
3. 查看电脑新 IP（`ifconfig`）
4. 更新小程序配置

## 📋 排查清单

- [ ] 端口号正确（3000 不是 300）
- [ ] 手机和电脑在同一 Wi-Fi
- [ ] macOS 防火墙允许 Node 或已关闭
- [ ] 路由器没有 AP 隔离
- [ ] 尝试 ngrok 内网穿透
